# Итоговый проект по курсу «Инженерия Искусственного Интеллекта»

В этой папке находится итоговый мини-проект по курсу.  
Проект должен демонстрировать применение методов и инструментов инженерии ИИ: работу с данными, модели, пайплайны, сервис, эксперименты и (по возможности) воспроизводимость.

## 1. Паспорт проекта

- **Название проекта:** Прогноз оттока клиентов телеком-оператора (Customer Churn Prediction)
- **Автор:** Соловьев Кирилл Дмитриевич
- **Группа:** ЭФБО-04-24
- **Контакт:** @kirill994

- **Краткое описание (2-4 предложения):**  
 >  Проект решает задачу бинарной классификации — предсказание, уйдёт ли клиент из телекоммуникационной компании в ближайший месяц.
> Используется открытый датасет Telco Customer Churn. Проведён разведочный анализ, обучены несколько моделей (логистическая регрессия, случайный лес, XGBoost, MLP), выполнен подбор гиперпараметров.
 > Результат – REST API сервис на FastAPI, который по признакам клиента возвращает вероятность оттока и категорию риска. Все эксперименты логируются с помощью MLflow.

## 2. Структура проекта

Проект организован в следующей структуре:

  - exp02_baseline_models.ipynb # Базовые модели
  - exp03_neural_baseline.ipynb # Нейросетевой базлайн
  - exp04_model_tuning.ipynb # Улучшение моделей
  - exp05_experiment_tracking.ipynb # Эксперименты и трекинг
- `src/` # основной код
  - model_loader.py # загрузка модели и препроцессора
  - service.py # FastAPI сервис
- `data/` # исходные данные (CSV)

  - preprocessor.joblib

## 3. Требования и установка

### 3.1. Требования

- Python `>= 3.10`
- pip и виртуальное окружение (рекомендуется)

### 3.2. Установка окружения

```bash
# Перейти в папку проекта
cd project

# Создать виртуальное окружение
python -m venv .venv

# Активировать окружение:
# Windows:
.venv\Scripts\activate
# Linux / macOS:
source .venv/bin/activate

# Установить зависимости
pip install --upgrade pip
pip install -r requirements.txt
```

### 4.2. Запуск сервиса (API/веб-интерфейс)

```bash
cd project
source .venv/bin/activate
python -m src.service
```

Сервис запустится на `http://localhost:8000`.
Доступные эндпоинты:

- GET /health – проверка работоспособности (возвращает {"status": "ok"})

- POST /predict – предсказание оттока.
Принимает JSON с признаками клиента (см. описание в Swagger).

- GET /metrics – служебные метрики (количество запросов, ошибок).

Swagger UI доступен по адресу `http://localhost:8000/docs`.

Пример запроса через curl:

## 8. Ограничения и дальнейшая работа

### Текущие ограничения:

- Модель обучена на исторических данных и не учитывает сезонность или макроэкономические факторы.
- API не имеет авторизации и не сохраняет историю запросов.
- Для высоких нагрузок требуется масштабирование (балансировка, кэширование).

### Направления для улучшения:

- Добавить контейнеризацию (Docker, docker-compose).
- Внедрить онлайн‑обучение с обратной связью по факту оттока.
- Расширить мониторинг (алерты при падении качества).
- Добавить асинхронную обработку и кэш для повторяющихся запросов.
