## 5. Analysis

### 5.1 Сравнение алгоритмов (важные наблюдения)

- KMeans "ломается" на Dataset B из-за нелинейной структуры данных

- Масштабирование критично для KMeans и метрик расстояния
- Категориальные признаки требуют аккуратного кодирования

- Вывод: KMeans устойчив на этом датасете благодаря чёткой кластерной структуре

## 6. Conclusion

1. Масштабирование признаков критически важно для корректной работы distance-based алгоритмов
2. KMeans плохо справляется с нелинейными структурами и данными разной плотности
3. DBSCAN хорош для данных с шумом и кластерами разной плотности, но требует аккуратного подбора параметров
4. Иерархическая кластеризация показывает стабильные результаты на разных типах данных
5. Метрики качества (silhouette, DB, CH) дают разные оценки, важно смотреть их в комплексе
6. Визуализация через PCA(2D) помогает понять структуру данных, но может искажать многомерные отношения
7. Категориальные признаки требуют специальной обработки (encoding)
8. Проверка устойчивости результатов важна для доверия к полученным кластерам
